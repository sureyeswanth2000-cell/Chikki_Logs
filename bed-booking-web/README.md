# Chikki Beds Web MVP

Website-first MVP for bed booking platform with three role views:
- Consumer
- Owner
- Superadmin

## Tech Stack
- Next.js (App Router, TypeScript)
- Tailwind CSS
- Firebase SDK (Firestore + Auth)

## Setup
1. Install dependencies:

```bash
npm install
```

2. Create env file from example:

```bash
copy .env.example .env.local
```

3. Fill Firebase values in .env.local.

For city seeding, also provide Firebase Admin credentials in .env.local:

- FIREBASE_ADMIN_PROJECT_ID
- FIREBASE_ADMIN_CLIENT_EMAIL
- FIREBASE_ADMIN_PRIVATE_KEY

Note: Keep newline escapes in FIREBASE_ADMIN_PRIVATE_KEY as \n.

4. Start development server:

```bash
npm run dev
```

Open http://localhost:3000

## Firestore Seed (Pilot Cities)
Run this command after .env.local is configured:

```bash
npm run seed:cities
```

This seeds 24 pilot cities into collection `cities`.

## Superadmin Setup (Backend)

1. Bootstrap first superadmin (one-time) using Admin SDK script:

```bash
npm run seed:superadmin -- --phone=+919876543210
```

You can also target by uid:

```bash
npm run seed:superadmin -- --uid=<firebase_uid>
```

2. Deploy callable function for future role changes:

```bash
firebase deploy --only functions:setUserRole
```

3. After bootstrap, superadmin can assign roles from backend callable `setUserRole`.

Credentials note for this app:
- Login method is Phone OTP.
- The script prints the phone for the promoted account.
- OTP is generated by Firebase at login time (no static password in this app).

## Locked Business Rules in MVP
- Cancellation allowed only within 15 minutes.
- Aadhaar required only at booking step.
- Advance payment INR 100.
- Final total shown to user.
- Payment gateway integration in later phase.

## Completed Features (Non-Payment)
- Role-based auth and protected routes (consumer/owner/superadmin)
- Owner inventory CRUD (properties, rooms, beds)
- Bed block management with full-block option
- Consumer city search, filters, and booking flow
- Booking conflict checks and short booking lock to reduce double booking
- Superadmin live KPI cards from Firestore data

## Firestore Security Rules
Rules file is available at:
- firestore.rules

Deploy rules with Firebase CLI:

```bash
firebase deploy --only firestore:rules
```
